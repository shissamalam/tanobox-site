Proprietary Engine

A calculation engine built for architects. Not accountants.

Generic FP&A tools model revenue as a single line item. Architecture firms have fundamentally different financial structures. TanoBox's proprietary engine is built for AEC-specific logic that no current competitor offers.

⚙
Custom Calculation Engine
Purpose-built TypeScript formula engine. DAG-based dependency resolution with iterative circular reference solving. Built with Claude Code. Zero third-party formula library dependencies. No GPL licensing exposure.
🗄
PostgreSQL + Next.js Stack
Full 3-statement modeling stack. NUMERIC(20,4) precision for all financial values. Never floating-point. Drizzle ORM, tRPC, shadcn/ui, AG Grid Community (MIT). Self-hosted, all data on your machine.
🪟
Windows Native Delivery
Ships as a pre-configured mini PC or a single PowerShell installer. WSL2 + Ubuntu + PM2. Opens in a browser tab. No cloud account, no Docker knowledge, no IT department required.

Risk Register

Known risks. Documented responses.

Five highest-priority risks from the full business plan, each with a concrete mitigation rule.

Calculation engine produces incorrect financial outputs
Critical
50+ unit tests before any customer sees the product. Balance sheet must balance to zero. CFS must reconcile in all scenarios. Explicit EULA disclaimer. "Verify with your CPA" language in product UI.
Support Circuit Breaker: manual support exceeds 10 hrs/week
New Rule
If manual support exceeds 10 hours in any week, all outbound sales outreach pauses immediately. Redirect all available time to improving self-service recovery tools. Resume outreach only when weekly support hours drop below 6.
Windows Update breaks WSL2 on customer machine
High
Repair script on desktop at install time. Weekly WSL2 auto-export to C:\TanoBox\backup.tar.gz. Group Policy defers feature updates 365 days. One-click recovery in 3 minutes.
QuickBooks changes API requirements
Medium
CSV Universal Import is the primary fallback — works with any accounting system, no Intuit approval required. QB Desktop SOAP integration as secondary fallback. Both are already built and tested.
Founder capacity collapse across build, sell, and support
High
Hard cap at 5 customers until support model is proven. Support Circuit Breaker enforces this automatically. Partners not activated until solo validation produces 5 paying customers with documented close rates.
A competitor launches a one-time pricing tier
Low Risk
Defense is deep AEC vertical positioning and proprietary engine features — backlog burn, billing schedules, utilization modeling — that a generic competitor cannot replicate by changing their pricing model.
